# <a name="synchronize-project-estimates-directly-from-project-service-automation-to-finance-and-operations"></a>Жоба болжамдарын тікелей Project Service Automation бағдарламасынан Finance and Operations бағдарламасына тікелей синхрондау

[!include[banner](../includes/banner.md)]

Бұл тақырыпта үлгілер және жобаның сағаттық болжамдары мен жобаның шығыс болжамдарын Dynamics 365 Project Service Automation бағдарламасынан Dynamics 365 Finance жүйесіне тікелей синхрондау үшін пайдаланылатын негізгі тапсырмалар сипатталады.

> [!NOTE]
> - Жоба тапсырмасын біріктіру, шығыс транзакция санаттары, сағаттық болжамдар, шығыс болжамдар және функционалдылықты құлыптау 8.0. нұсқасында қолжетімді болады.
> - Нақты көрсеткіштерді біріктіру 8.0.1 немесе одан кейінгі нұсқада қолжетімді.

## <a name="data-flow-for-project-service-automation-to-finance"></a>Project Service Automation және Finance бағдарламасындағы деректер ағыны

Project Service Automation және Finance бағдарламасын біріктіру шешімі деректерді біріктіру мүмкіндігін Project Service Automation және Finance мысалдары бойынша деректерді синхрондау үшін пайдаланады. Деректерді біріктіру функциясымен қолжетімді болатын біріктіру үлгілері жобаның сағаттық болжамдары мен жобаның шығыс болжамдары туралы деректер ағынын Project Service Automation бағдарламасынан Finance бағдарламасына жіберуге мүмкіндік береді.

Келесі суретте деректердің Project Service Automation және Finance арасында синхрондалу жолы көрсетілген.

[![Project Service Automation бағдарламасын Finance бағдарламасымен біріктіруге арналған деректер ағыны.](./media/ProjectEstimatesFlow.png)](./media/ProjectEstimatesFlow.png)

## <a name="project-hour-estimates"></a>Жобаның сағаттық болжамдары

### <a name="template-and-tasks"></a>Үлгі және тапсырмалар

Қолжетімді үлгілерге қатынасу үшін Microsoft Power Apps басқару орталығында **Жобалар** опциясын таңдап, содан соң жоғарғы оң жақтағы бұрышта жалпы үлгілерді таңдау үшін **Жаңа жоба** опциясын таңдаңыз.

Келесі үлгілер және негізгі тапсырмалар жобаның сағаттық болжамдарын Project Service Automation бағдарламасынан Finance бағдарламасына синхрондау үшін пайдаланылады.

- **Деректерді біріктіру функциясындағы үлгі атауы:** жобаның сағаттық болжамдары (PSA бағдарламасынан Fin және Ops бағдарламасына)
- **Жобадағы тапсырмалардың атауы**

    - Транзакция қатынастары
    - Шығыс болжамдары

### <a name="entity-set"></a>Нысан жиынтығы

| Project Service Automation | Қаржы                                       |
|----------------------------|-----------------------------------------------|
| Жоба тапсырмалары              | Жобаның сағаттық болжамдары үшін біріктіру нысаны |

### <a name="entity-flow"></a>Нысан ағыны

Жобаның сағаттық болжамдары Project Service Automation бағдарламасында басқарылады және олар Finance бағдарламасында жобаның сағаттық болжамдары ретінде синхрондалады.

### <a name="prerequisites"></a>Алғышарттар

Жобаның сағаттық болжамдарын синхрондау орындалмас бұрын, жобаларды, жоба тапсырмаларын және жобаның шығыс транзакциясының санаттарын синхрондау қажет.

### <a name="power-query"></a>Power Query

Жобаның сағаттық болжамдары үлгісінде, осы тапсырмаларды орындау үшін Excel бағдарламасына арналған Microsoft Power Query қондырмасын пайдалану қажет.

- Біріктіру жаңа сағаттық болжамдарды жасаған кезде пайдаланылатын әдепкі болжам үлгісінің идентификаторын орнатыңыз.
- Сағаттық болжамдарға біріге алмайтын тапсырмадағы кез келген ресурсқа арналған жазбаларды сүзгілеңіз.
- Кез келген бос транзакция санатының жолдарын сүзгілеңіз. Бұл тапсырманы орындамау қате сағаттық болжамдарға әкелуі мүмкін.

#### <a name="set-the-default-forecast-model-id"></a>Әдепкі болжам үлгісінің идентификаторын орнату

Үлгідегі әдепкі болжам идентификаторын орнату үшін, **Карта** көрсеткісін салыстыруды ашу үшін басыңыз. Содан кейін **Кеңейтілген сұрау және сүзгілеу** сілтемесін таңдаңыз.

- Егер әдепкі "Жобаның сағаттық болжамдары" (PSA бағдарламасынан Fin және Ops бағдарламасына) үлгісін пайдалансаңыз, **Қолданылған қадамдар** тізіміндегі **Енгізілген шарт** параметрін таңдаңыз. **Функция** жазбасында, біріктірумен пайдаланылуы керек болжам үлгісінің идентификатор атауы бар **O\_болжам** бағанын ауыстырыңыз. Әдепкі үлгіде демо деректерінен алынған болжам үлгісінің идентификаторы бар.
- Егер жаңа үлгі жасасаңыз, осы бағанды қосу қажет. Power Query қондырмасында, **Шартты бағанды қосу** параметрін таңдап, **ModelID** сияқты жаңа бағанға арналған атау енгізіңіз. Бағанның шартын енгізіңіз, егер жоба тапсырмасы нөлге тең болмаса, онда \<enter the forecast model ID\>; басқасы нөл.

#### <a name="filter-out-resource-specific-records"></a>Ресурсқа арналған жазбаларды сүзгілеу

Жобаның сағаттық болжамдары (PSA бағдарламасынан Fin және Ops бағдарламасына) үлгісінде кез келген ресурсқа арналған жазбаларды жоятын әдепкі сүзгі бар. Егер өз үлгіңізді жасасаңыз, онда осы сүзгіні қосу қажет. Тек **Жалған** жазбалар қамтылатындай етіп **msdyn\_islinetask** бағанында сүзгілеу үшін **Кеңейтілген сұрау және сүзгілеу** сілтемесін таңдаңыз.

#### <a name="filter-out-empty-transaction-category-rows"></a>Бос транзакция санатының жолдарын сүзгілеу

Бос транзакция санаттары бар кез келген жолдарды жою үшін сүзгі қосу қажет. Бұл тапсырма әдепкі үлгіні пайдалануыңызға немесе жеке үлгіні жасауыңызға қарамастан қажет. Бұл сүзгі қорытынды жолдарды сағаттық болжамдардың Finance бағдарламасында қате болуына әкелуі мүмкін Project Service Automation бағдарламасынан жояды. **Кеңейтілген сұрау және сүзгілеу** сілтемесін **msdyn\_transactioncategory\_value** бағанындаға бос жазбаларды сүзгілеу үшін таңдаңыз.

### <a name="template-mapping-in-data-integration"></a>Деректерді біріктіруде үлгіні салыстыру

Келесі суретте деректерді біріктіру қызметіндегі үлгі тапсырмасын салыстыру мысалы көрсетіледі. Картада Project Service Automation бағдарламасынан Finance бағдарламасына синхрондалатын өріс туралы ақпарат көрсетілген.

[![Деректерді біріктіруде үлгі тапсырманы салыстыру.](./media/ProjectHourEstimatesMapping.jpg)](./media/ProjectHourEstimatesMapping.jpg)

## <a name="project-expense-estimates"></a>Жобаның шығыс болжамдары

### <a name="template-and-tasks"></a>Үлгі және тапсырмалар

Келесі үлгі және негізгі тапсырмалар жобаның шығыс болжамдарын Project Service Automation бағдарламасынан Finance бағдарламасына синхрондау үшін пайдаланылады.

- **Деректерді біріктіру функциясындағы үлгі атауы:** жобаның шығыс болжамдары (PSA бағдарламасынан Fin және Ops бағдарламасына)
- **Жобадағы тапсырмалардың атауы**

    - Транзакция қатынастары 
    - Шығыс болжамдары

### <a name="entity-set"></a>Нысан жиынтығы

| Project Service Automation | Қаржы                                                  |
|----------------------------|----------------------------------------------------------|
| Транзакция байланыстары    | Жобаның транзакция қатынастарына арналған біріктіру нысаны |
| Болжам жолдары             | Жобаның шығыс болжамдары үшін біріктіру нысаны         |

### <a name="entity-flow"></a>Нысан ағыны

Жобаның шығыс болжамдары Project Service Automation бағдарламасында басқарылады және олар Finance бағдарламасында жобаның шығыс болжамдары ретінде синхрондалады.

### <a name="prerequisites"></a>Алғышарттар

Жобаның шығыс болжамдарын синхрондау орындалмас бұрын, жобаларды, жоба тапсырмаларын және жобаның шығыс транзакциясының санаттарын синхрондау қажет.

### <a name="power-query"></a>Power Query

Жобаның шығыс болжамдары үлгісінде, келесі тапсырмаларды орындау үшін Excel бағдарламасына арналған Microsoft Power Query қондырмасын пайдалану қажет.

- Тек шығыс болжамының жол жазбаларын қамту үшін сүзгілеңіз.
- Біріктіру жаңа сағаттық болжамдарды жасаған кезде пайдаланылатын әдепкі болжам үлгісінің идентификаторын орнатыңыз.
- Есеп-шот түрлерін түрлендіріңіз.
- Транзакция түрлерін түрлендіріңіз.

#### <a name="filter-to-include-only-expense-estimate-lines"></a>Тек шығыс болжамының жолдарын қамту үшін сүзгілеу

Жобаның шығыс болжамдары (PSA бағдарламасынан Fin және Ops бағдарламасына) үлгісінде біріктірудегі шығыс жолдарды қамтитын әдепкі сүзгі бар. Егер өз үлгіңізді жасасаңыз, онда осы сүзгіні қосу қажет. **Транзакция қатынастары** тапсырмасын таңдаңыз, содан кейін салыстыруды ашу үшін **Карта** көрсеткісін басыңыз. **Кеңейтілген сұрау және сүзгілеу** сілтемесін таңдаңыз. **msdyn\_transactiontype1** бағанын тек **msdyn\_estimateline** бағанын қамтитындай етіп сүзгілеңіз.

#### <a name="set-the-default-forecast-model-id"></a>Әдепкі болжам үлгісінің идентификаторын орнату

Үлгідегі әдепкі болжам идентификаторын жаңарту үшін, **Шығыс болжамдары** тапсырмасын таңдаңыз, содан кейін салыстыруды ашу үшін **Карта** көрсеткісін басыңыз. **Кеңейтілген сұрау және сүзгілеу** сілтемесін таңдаңыз.

- Егер әдепкі жобаның шығыс болжамдары (PSA бағдарламасынан Fin және Ops бағдарламасына) үлгісін пайдалансаңыз, Power Query қондырмасында, алдымен **Қолданылған қадамдар** бөлімінен **Енгізілген шарт** параметрін таңдаңыз. **Функция** жазбасында, біріктірумен пайдаланылуы керек болжам үлгісінің идентификатор атауы бар **O\_болжам** бағанын ауыстырыңыз. Әдепкі үлгіде демо деректерінен алынған болжам үлгісінің идентификаторы бар.
- Егер жаңа үлгі жасасаңыз, осы бағанды қосу қажет. Power Query қондырмасында, **Шартты бағанды қосу** параметрін таңдап, **ModelID** сияқты жаңа бағанға арналған атау енгізіңіз. Бағанның шартын енгізіңіз, егер болжам жолының идентификаторы нөлге тең болмаса, онда \<enter the forecast model ID\>; басқасы нөл.

#### <a name="transform-the-billing-types"></a>Есеп-шот түрлерін түрлендіру

Жобаның шығыс болжамдары (PSA бағдарламасынан Fin және Ops бағдарламасына) үлгісі біріктіру кезінде Project Service Automation бағдарламасынан алынған есеп-шот түрлерін түрлендіру үшін пайдаланылатын шартты бағанды қамтиды. Егер жеке үлгіңізді жасасаңыз, шартты бағанды қосу қажет. **Кеңейтілген сұрау және сүзгілеу** сілтемесін, содан кейін **Шартты бағанды қосу** параметрін таңдаңыз. Жаңа бағанға арналған **BillingType** сияқты атауды енгізіңіз. Содан кейін келесі шартты енгіңіз:

Егер **msdyn\_billingtype** = 192350000 болса, онда **Ақысыз**  
басқасы **msdyn\_billingtype** = 192350001 болса, онда **Ақылы**  
басқасы **msdyn\_billingtype** = 192350002 болса, онда **Қосымша**  
басқасы **Қолжетімді емес**

#### <a name="transform-the-transaction-types"></a>Транзакция түрлерін түрлендіру

Жобаның шығыс болжамдары (PSA бағдарламасынан Fin және Ops бағдарламасына) үлгісі біріктіру кезінде Project Service Automation бағдарламасынан алынған транзакция түрлерін түрлендіру үшін пайдаланылатын шартты бағанды қамтиды. Егер жеке үлгіңізді жасасаңыз, шартты бағанды қосу қажет. **Кеңейтілген сұрау және сүзгілеу** сілтемесін, содан кейін **Шартты бағанды қосу** параметрін таңдаңыз. Жаңа бағанға арналған **TransactionType** сияқты атауды енгізіңіз. Содан кейін келесі шартты енгіңіз:

Егер **msdyn\_transactiontypecode** = 192350000 болса, онда **Шығын**  
басқасы **msdyn\_transactiontypecode** = 192350005 болса, онда **Сатылым**  
басқасы **бос**
