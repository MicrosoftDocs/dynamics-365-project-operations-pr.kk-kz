# <a name="troubleshoot-working-in-the-task-grid"></a>Тапсырмалар торында жұмыс істеу кезінде ақаулықтарды жою 


_**Келесіге қолданылады:** ресурсқа/биржадан тыс негізделген сценарийлерге арналған Project Operations, жеңілдетілген орналастыру - проформа есеп‑шотын ұсыну мәмілесі, Project for the Web_

Dynamics 365 Project Operations бағдарламасы пайдаланатын тапсырмалар торы — Microsoft Dataverse ішіне орналастырылған iFrame. Осы пайдалану нәтижесінде аутентификация мен авторизацияның дұрыс жұмыс істеуін қамтамасыз ету үшін арнайы талаптар орындалуы керек. Бұл тақырыпта торды көрсету немесе жұмыс декомпозициясының құрылымындағы (WBS) тапсырмаларды басқару әрекеттеріне әсер етуі мүмкін жалпы мәселелер сипатталады.

Жалпы мәселелерге мыналар жатады:

- Тапсырма торындағы **Тапсырма** қойыншасы бос.
- Жобаны ашқан кезде жоба жүктелмейді және пайдаланушы интерфейсі (UI) санағышта тұрып қалады.
- **Project for the Web** бағдарламасына арналған артықшылықтарды басқару.
- Тапсырманы жасау, жаңарту немесе жою кезінде өзгерістер сақталмайды.

## <a name="issue-the-task-tab-is-empty"></a>Мәселе: тапсырмалар қойыншасы бос

### <a name="mitigation-1-enable-cookies"></a>1-жеңілдету жолы: cookie файлдарын қосу

Project Operations бағдарламасы жұмыс декомпозициясының құрылымын көрсету үшін үшінші тараптың cookie файлдарын қосуды талап етеді. Үшінші тарап cookies файлдары қосылмаған кезде, тапсырмаларды көрудің орнына, **Жоба** бетіндегі **Тапсырмалар** қойыншасын таңдағанда, бос бетті көресіз.

Microsoft Edge немесе Google Chrome шолғыштарында келесі процедуралар үшінші тарап cookies файлдарын қосу үшін шолғыш параметрін жаңарту жолдарын көрсетеді.

#### <a name="microsoft-edge"></a>Microsoft Edge

1. Edge шолғышын ашыңыз.
2. Үстіңгі оң жақ бұрыштан **көп нүкте** (…) белгішесін таңдап, **Параметрлер** опциясын таңдаңыз.
3. **Cookie файлдары және сайт рұқсаттары** бөлімінде **Cookie файлдары және сайт туралы деректер** опциясын таңдаңыз.
4. **Үшінші тарап cookie файлдарын бұғаттау** функциясын өшіріңіз.
5. Браузерді жаңартыңыз. 

#### <a name="google-chrome"></a>Google Chrome

1. Chrome шолғышын ашыңыз.
2. Үстіңгі оң жақ бұрыштан үш тік нүктені таңдап, содан кейін **Параметрлер** опциясын таңдаңыз.
3. **Құпиялылық және қауіпсіздік** бөлімінде **Cookie файлдары және басқа сайт туралы деректер** опциясын таңдаңыз.
4. **Барлық cookie файлдарына рұқсат беру** опциясын таңдаңыз.
5. Браузерді жаңартыңыз. 

> [!NOTE]
> Егер үшінші тарапты cookie файлдарын бұғаттасаңыз, тіпті, ерекшеліктер тізімінде сайтқа рұқсат берілсе де басқа сайттардағы барлық cookie деректері мен сайт деректері бұғатталады.

### <a name="mitigation-2-validate-the-pex-endpoint-has-been-correctly-configured"></a>2-жеңілдету жолы: PEX соңғы нүктесінің дұрыс конфигурацияланғанын тексеру

Project Operations бағдарламасы жоба параметрінің PEX соңғы нүктесіне сілтемесін талап етеді. Бұл соңғы нүкте жұмыс декомпозициясының құрылымын көрсету үшін пайдаланылатын қызметпен байланысу үшін қажет. Егер параметр қосылмаған болса, сізге "Жоба параметрі жарамды емес" деген қате жіберіледі. PEX соңғы нүктесін жаңарту үшін келесі қадамдарды орындаңыз.

1. **Жоба параметрлері** бетіне **PEX соңғы нүктесі** өрісін қосыңыз.
2. Қолданылатын өнім түрін анықтаңыз. Бұл мән PEX соңғы нүктесі орнатылған кезде қолданылады. Шығару кезінде өнім түрі PEX соңғы нүктесінде анықталған. Сол мәнді сақтаңыз.
3. Өрісті келесі мәнмен жаңартыңыз: `https://project.microsoft.com/<lang>/?org=<cdsServer>#/taskgrid?projectId=<id>&type=2`. Келесі кестеде өнім түріне байланысты пайдаланылуы тиіс түр параметрі беріледі.

      | **Өнім түрі**                     | **Параметр түрі** |
      |--------------------------------------|--------------------|
      | Әдепкі ұйымдағы Project for the Web   | түрі=0             |
      | CDS деп аталатын ұйымдағы Project for the Web | түрі=1             |
      | Project Operations                   | түрі=2             |

4. **Жоба параметрлері** бетінен өрісті жойыңыз.

## <a name="issue-the-project-doesnt-load-and-the-ui-is-stuck-on-the-spinner"></a>Мәселе: жоба жүктелмейді және UI санағышта тұрып қалады

Аутентификация мақсаттарында тапсырмалар торы жүктелуі үшін қалқымалы терезелерді қосу қажет. Егер қалқымалы терезелер қосылмаса, экран жүктеу санағышында тұрып қалады. Келесі суретте мекенжай жолағында бұғатталған қалқымалы терезе белгісі бар URL мекенжайы көрсетіледі, оның нәтижесінде санағыш бетті жүктей алмай, тұрып қалады. 

   ![Тұрып қалған санағыш және бұғатталған қалқымалы терезе.](media/popupsblocked.png)

### <a name="mitigation-1-enable-pop-ups"></a>1-жеңілдету жолы: қалқымалы терезелерді қосу

Егер жобаңыз санағышта тұрып қалса, онда қалқымалы терезелер қосылмай тұруы мүмкін.

#### <a name="microsoft-edge"></a>Microsoft Edge

Edge браузерінде қалқымалы терезелерді қосудың екі жолы бар.

1. Edge браузерінде, браузердің жоғарғы оң жағындағы хабарландыруды таңдаңыз.
2. Нақты Dataverse ортасынан шығатын **Қалқымалы терезелер мен қайта бағыттауларға әрқашан рұқсат беру** опциясын таңдаңыз.
 
     ![Қалқымалы терезелер бұғатталған.](media/enablepopups.png)

Не болмаса, келесі қадамдарды орындауға болады:

1. Edge шолғышын ашыңыз.
2. Жоғарғы оң жақ бұрышта **көп нүктені** (...) таңдап, содан кейін **Параметрлер** > **Сайт рұқсаттары** > **Қалқымалы терезелер мен қайта бағыттаулар** опциясын таңдаңыз.
3. **Қалқымалы терезелер мен қайта бағыттаулар** опциясын қалқымалы терезелерді бұғаттау үшін өшіріңіз немесе құрылғыда қалқымалы терезелерге рұқсат беру үшін қосыңыз.
4. Қалқымалы терезелерді қосқаннан кейін, браузерді жаңартыңыз. 

#### <a name="google-chrome"></a>Google Chrome
1. Chrome шолғышын ашыңыз.
2. Қалқымалы терезелер бұғатталған бетке өтіңіз.
3. Мекенжай жолағында **Қалқымалы терезе бұғатталды** опциясын таңдаңыз.
4. Көру керек қалқымалы терезеге өтетін сілтемені таңдаңыз.
5. Қалқымалы терезелерді қосқаннан кейін, браузерді жаңартыңыз. 

> [!NOTE]
> Сайттың қалқымалы терезелерін үнемі көру үшін **[сайттан] қалқымалы терезелер мен қайта бағыттауға әрқашан рұқсат беру** опциясын, содан кейін **Дайын** опциясын таңдаңыз.

## <a name="issue-3-administration-of-privileges-for-project-for-the-web"></a>3-мәселе: Project for the Web бағдарламасына арналған артықшылықтарды басқару

Project Operations бағдарламасы сыртқы жоспарлау қызметіне негізделген. Бұл қызмет WBS әдісіне қатысты нысандарды оқуға және жазуға мүмкіндік беретін бірнеше рөлдің пайдаланушыға тағайындалуын талап етеді. Бұл нысандар жобалық тапсырмаларды, ресурстарды тағайындауды және тапсырма тәуелділіктерін қамтиды. Егер пайдаланушы **Тапсырмалар** қойыншасына өткенде WBS көрсете алмаса, бұл **Project Operations** бағдарламасы үшін **Жоба** опциясының іске қосылмауынан болуы мүмкін. Пайдаланушыға қауіпсіздік рөлі қатесі немесе кіруден бас тартуға байланысты қате жіберілуі мүмкін.

### <a name="mitigation-1-validate-the-application-user-and-end-user-security-roles"></a>1-жеңілдету жолы: бағдарлама пайдаланушысы мен соңғы пайдаланушының қауіпсіздік рөлдерін тексеру

1. **Параметр** > **Қауіпсіздік** > **Пайдаланушылар** > **Бағдарлама пайдаланушылары** тармағына өтіңіз.  

   ![Бағдарламаны оқу құралы.](media/applicationuser.jpg)
   
2. Төмендегілерді тексеру үшін бағдарлама пайдаланушысының жазбасын екі рет түртіңіз:

     - Пайдаланушы жобаға қол жеткізе алады. Оны пайдаланушыда **Жоба менеджері** қауіпсіздік рөлі бар екенін тексеру арқылы орындауға болады.
     - Microsoft Project бағдарламасы пайдаланушысы бар және дұрыс конфигурацияланған.
 
3. Егер бұл пайдаланушы жоқ болса, жаңа пайдаланушы жазбасын жасаңыз. 
4. **Жаңа пайдаланушылар** опциясын таңдап, енгізу пішінін **Бағдарлама пайдаланушысы** пішініне өзгертіп, содан кейін **Бағдарлама идентификаторы** параметрін қосыңыз.

   ![Бағдарлама пайдаланушысы туралы мәліметтер.](media/applicationuserdetails.jpg)


## <a name="issue-4-changes-arent-saved-when-you-create-update-or-delete-a-task"></a>4-мәселе: тапсырманы жасау, жаңарту немесе жою кезінде өзгерістер сақталмайды

WBS әдісіне бір немесе бірнеше жаңартуды енгізгенде, өзгерістер сәтсіз жасалып, сақталмайды. Кесте торында "Сіз жасаған соңғы өзгерісті сақтау мүмкін болмады" деген хабармен бірге қате пайда болады.

### <a name="mitigation-1-validate-the-license-assignment"></a>1-жеңілдету жолы: лицензия тағайындауын растау

1. Пайдаланушыға дұрыс лицензия берілгенін және қызметтің лицензия туралы егжей-тегжейлі жоспарлау қызметінде қосылғанын тексеріңіз.  
2. Пайдаланушы **project.microsoft.com** сайтын аша алатынын тексеріңіз.
    
### <a name="mitigation-2-validation-configuration-of-the-project-application-user"></a>2-жеңілдету жолы: жоба бағдарламасы пайдаланушысының конфигурациясын тексеру
1. Жоба бағдарламасы пайдаланушысының жазбасы жасалғанын тексеріңіз.
2. Пайдаланушыға келесі қауіпсіздік рөлдерін қолданыңыз:
  
  - Dataverse пайдаланушысы немесе негізгі пайдаланушысы
  - Project Operations жүйесі
  - Жоба жүйесі
  - Project Operations қос жазу жүйесі. Бұл рөл Project Operations бағдарламасының ресурс/биржадан тыс негіздегі сценарийлерін орналастыру үшін қажет.
