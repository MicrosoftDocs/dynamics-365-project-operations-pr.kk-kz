# <a name="create-a-work-breakdown-structure-wbs"></a>Жұмыс декомпозициясының құрылымын жасау (WBS)

Жоба кестесінде орындалатын жұмыс, жұмысты орындайтын ресурстар және жұмыстың орындалып бітуі керек мерзімі көрсетіледі. Кесте жобаны уақытылы жеткізумен байланысты барлық жұмысқа әсер етеді. Dynamics 365 Project Operations бағдарламасында жоба кестесін келесі жолмен жасайсыз:

  - Жұмысты басқарылатын тапсырмаларға бөлу.
  - Әр тапсырманы орындауға кететін уақытты болжау.
  - Тапсырманың тәуелділіктерін орнату.
  - Тапсырманың ұзақтығын орнату.
  - Тапсырмаларды орындайтын жалпы ресурстарды болжау. 

Жоба кестесі **Жоба** бетінің **Кесте** қойыншасында жасалады.

## <a name="tasks"></a>Тапсырмалар

Жоба кестесін құрудың алғашқы қадамы жұмысты басқарылатын бөліктерге бөлу болып табылады. Project Operations бағдарламасындағы кесте келесі мүмкіндіктерге қолдау көрсетіледі:

- Қорытынды немесе контейнер тапсырмалары
- Соңғы түйін тапсырмалары

### <a name="summary-tasks"></a>Жиынтық тапсырмалар

Жиынтық тапсырмалар басқа жиынтық тапсырмаларды немесе соңғы түйін тапсырмаларын сақтай алады. Олардың жұмыс күші немесе өзіндік құны жоқ. Оның орнына, олардың жұмыс қарқындылығы мен құны — бұл контейнер тапсырмаларының жұмыс қарқындылығы мен құнының жиынтығы. Жиынтық тапсырманың басталу күні контейнер тапсырмаларының басталу күні, ал аяқталу күні контейнер тапсырмаларының аяқталу күні болып табылады. Жиынтық тапсырманың атауын өзгертуге болады, бірақ күш, күндер және ұзақтық сияқты жоспарлау қасиеттерін өзгерту мүмкін емес. Егер жиынтық тапсырманы жойсаңыз, оның барлық контейнер тапсырмаларын да жоясыз.

### <a name="leaf-node-tasks"></a>Соңғы түйін тапсырмалары

Соңғы түйін тапсырмалары жобадағы толық мәліметті жұмысты көрсетеді. Мұнда болжалды талпыныс, ресурстар, жоспарлы басталу және аяқталу күндері, ұзақтығы бар.

## <a name="create-a-task-hierarchy"></a>Тапсырма иерархиясын жасау

### <a name="add-a-task"></a>Тапсырма қосу

Бір немесе бірнеше тапсырма қосу үшін келесі қадамдарды орындаңыз.

1. **Жобалар** бетіне өтіп, кесте жасағыңыз келетін жоба жазбасын таңдаңыз және ашыңыз. 
2. **Тапсырмалар** қойыншасын таңдаңыз. 
3. **Жаңа тапсырма қосу** пәрменін таңдаңыз, тапсырманың атын енгізіп, Enter пернесін басыңыз.
2. Тапсырмалардың толық тізімі болғанша тағы бір тапсырма атын енгізіп, Enter пернесін қайтадан басыңыз.

### <a name="manage-hierarchy-of-a-task"></a>Тапсырманың иерархиясын басқарыңыз

Тапсырма жасалған кезде, ол тікелей оның үстіне қойылған тапсырманың қосалқы элементі болады. Тапсырманың кесте идентификаторы кейін жаңа басты элементінің кесте идентификаторына негізделетіндей және құрылымды нөмірлеу сызбасына сәйкес болатындай етіп қайта есептеледі. Негізгі тапсырма енді қорытынды тапсырма болып табылады. Сондықтан бұл қосалқы тапсырмаларының жиынтығына айналады. Тапсырма ілгерілетілген кезде, енді оның басты элементі болған тапсырманың еншілес элементі болмайды. Содан кейін кесте идентификаторы қайта есептеледі, осылайша ол тапсырманың жаңартылған тереңдігі мен иерархиядағы орнын көрсетеді. Алдыңғы басты тапсырманың қарқындылығы, құны және мерзімдері осы тапсырманы қамтымайтын етіп қайта есептеледі.

Тапсырманы туралау немесе ілгерілету үшін келесі қадамдарды орындаңыз.

1. **Жоба** бетіндегі, **Тапсырмалар** қойыншасындағы **Қорытынды тапсырмалар** пәрменіндегі тапсырма атауы бойынша үш тік нүктені таңдап, содан кейін **Ішкі тапсырма жасау** пәрменін таңдаңыз. 
2. Туралау немесе ілгерілету үшін тапсырманы таңдаңыз. Бірнеше тапсырманы таңдау үшін тапсырманы таңдап, Ctrl пернесін басып тұрыңыз, содан кейін қосымша тапсырмаларды таңдаңыз.
2. Тапсырмаларды астынан немесе қорытынды тапсырмалар астынан жылжыту үшін **Туралау** немесе **Ішкі тапсырманы жариялау** пәрменін таңдаңыз.

### <a name="move-tasks-up-and-down"></a>Тапсырмаларды жоғары және төмен жылжытыңыз

Тапсырмаларды жұмысты бөлу құрылымындағы кез келген деңгейге екі жолдың бірімен жылжытуға болады:

- Тағы бір тапсырманы таңдап, оны қалаған орынға сүйреңіз.
- Бір немесе бірнеше тапсырманы таңдаңыз, тінтуірдің оң жағын басып, **Кесу** пәрменін таңдаңыз, кестедегі тағайындау ұяшығын таңдаңыз, содан кейін тінтуірдің оң жағын басып, **Қою** пәрменін таңдаңыз.

## <a name="task-attributes"></a>Тапсырма төлсипаттары

Тапсырма атауы орындалуы керек жұмысты сипаттайды. Project Operations бағдарламасында тапсырмаға байланысты атрибуттар тапсырманың кестесін және оның кадрлық қажеттіліктерін сипаттайды.

## <a name="schedule-attributes"></a>Кесте төлсипаттары

**Қарқындылық**, **Басталу күні**, **Аяқталу күні** және **Ұзақтығы** атрибуттары тапсырманың кестесін анықтайды.

Келесі кестеде қосымша кесте төлсипаттары көрсетілген.

| **Соңғы көрсетілетін атау** | **Соңғы сипаттама** |
| --- | --- |
| Талпыныс аяқталды (сағаттар) | Сағат ішінде аяқталған тапсырма жұмысы. |
| Ұзақтық | Тапсырма ұзақтығын күнмен көрсетеді. |
| Жалпы талпыныс | Сағат ішіндегі тапсырманың жалпы жұмысы. |
| Аяқтау | Аяқталу күні мен уақыты. |
| % аяқталды | Орындалған тапсырманың пайыздық мәні. |
| Жоба контейнері | Әрбір себетте жеке баған болуы үшін тапсырма тақтасын себет бойынша топтастыруға болады. |
| Қалған талпыныс (сағаттар) | Сағат ішіндегі тапсырманың қалған жұмысы (сағаттар). |
| Бастау | Басталу күні мен уақыты. |
| Атауы | Тапсырманың атауы. |
| Идентификатор | Жұмыс декомпозициясының құрылымындағы тапсырма идентификаторы. |

Әкімші ретінде тапсырма нысанында реттелетін өрістерді анықтай аласыз. Алайда өрістер кесте торында көрсетілмейді. Реттелетін өрістерді көру үшін оларды **Жоба тапсырмасы** мәліметтер бетіне қосыңыз.

## <a name="staffing-attributes"></a>Материал төлсипаттары

Материал атрибуттарына кестедегі **Ресурстар** өрісі арқылы қол жеткізуге болады. Бұрыннан бар ресурсты іздей аласыз немесе **Жасау** параметрін таңдап, **Жылдам жасау** бөлігінде жаңа ресурс ретінде жоба тобының мүшесін қосыңыз.  Тапсырмалар торында, тақта көрінісінде немесе gantt ішінде ресурс таңдау құралын пайдаланып ресурсты іздеген кезде, іздеу бар жоба тобы мүшелерін немесе белсенді тапсырыс берілетін ресурстарды қайтарады.

**Рөл**, **Ресурс бірлігі** және **Орын атауы** өрістері тапсырманың материал талаптарын сипаттау үшін қолданылады. Осы материал төлсипаттары тапсырма кестесімен бірге осы тапсырманы орындау мақсатында қолжетімді ресурстарды табу үшін пайдаланылады.

   - **Рөл** : Тапсырманы орындау үшін қажетті ресурс түрін көрсетіңіз.,
   - **Ресурс бірлігі**: тапсырма ресурстары тағайындалуы керек бірлікті көрсетіңіз. Бұл атрибут ресурстың құны мен вексель мөлшерлемесі ресурс бірліктеріне негізделген болса, тапсырманың құны мен сатылым болжамына әсер етеді.
   - **Орын атауы**: жалпы ресурсқа атау енгізіңіз, ол бүкіл жұмысты орындайтын ресурстың толтырғыш рөлін атқарады.

**Ресурстар** өрісі табылған кезде әмбебап немесе аталған ресурстың орын атауын қамтиды.

**Санат** өрісі тапсырманы топтастыруға болатын жұмыстың әлдеқайда кең түрін көрсететін мәндерді қамтиды. Бұл өріс жоспарлауға немесе қызметкерлер құрамына әсер етпейді. Оның орнына өріс тек есеп беру үшін қолданылады.

## <a name="task-dependencies"></a>Тапсырма тәуелділіктері

Тапсырмалар арасында алдыңғы қарым-қатынастарды құру үшін Project Operations бағдарламасындағы кестені қолдануға болады. **Алдыңғы** өрісі тапсырма тәуелді болатын тапсырмаларды көрсету үшін бір немесе бірнеше мәнді пайдаланады. Алдыңғы мәндер тапсырмаға тағайындалған кезде, тапсырма барлық алдындағы тапсырмалар аяқталғаннан кейін ғана басталады. Тәуелділікке байланысты, тапсырманың жоспарланған басталу күні алдыңғы тапсырмалар аяқталған күнге келтіріледі.

Тапсырма режимі алдыңғы/тәуелді тапсырмалардың басталу және аяқталу күндеріне жасалған жаңартуларға әсер етпейді.

## <a name="understanding-the-impacts-of-duration-resource-calendars-and-project-calendars-on-tasks"></a>Ұзақтық, ресурс күнтізбелері және жоба күнтізбелерінің тапсырмаларға әсерін түсіну
Тапсырманың ұзақтығы тапсырманың басталу күні мен аяқталу күнінің басталу уақыты арасындағы жұмыс сағаттарының саны ретінде анықталады.   Веб жобасы ұзақтық өлшем бірліктерін келесідей анықтайды:

| **Ұзақтық өлшемі** | **Шама**|
|----------------------------------------------------|----------------------|
| Күніне сағат | 8 |
| Аптасына сағаттар |  40 |
| Айына күндер |  20 |

Тағайындалмаған тапсырмалар жобаның күнтізбесін пайдалану арқылы жоспарланады. Дегенмен, бастапқы ресурс тағайындалғанда, тапсырманың жоспарлауы ресурс күнтізбесіне сәйкес келетіндей жаңартылады. Тапсырмасы бар тапсырмаға кейінгі өзгертулер келесімен реттеледі [жоспарлау режимі](scheduling-modes.md) жобаның. Күнтізбелердің тапсырмаларға әсері туралы көбірек білу үшін қараңыз [Вебке арналған жобадағы ресурс күнтізбелері](https://techcommunity.microsoft.com/t5/project-blog/resource-calendars-in-project-for-the-web/ba-p/3269686) және [Тапсырмаларды бастау уақыттары және сіздің жобаларыңыз!](https://techcommunity.microsoft.com/t5/project-blog/task-start-times-amp-your-projects/ba-p/3269665)


## <a name="accessibility-and-keyboard-shortcuts"></a>Арнайы мүмкіндіктер және пернелер тіркесімдері

**Кесте** торы толығымен қолжетімді және оны экрандық диктор, JAWS немесе NVDA сияқты экранды оқу құралдарымен бірге пайдалануға болады. Көрсеткі пернелердің көмегімен тор аймағында жылжуыңызға болады (Microsoft Excel бағдарламасындағы сияқты), интерактивті пайдаланушы интерфейсі элементтерін шарлау үшін Tab пернесін қолдана аласыз, ал ашылмалы мәзірлерді таңдау және ашу үшін төменгі көрсеткі пернесін, Enter пернесін немесе бос орынды пайдалана аласыз.
