# <a name="actuals"></a>Қолда бар тауарлар 

_**Қолданылады:** Ресурс/биржадан тыс негіздегі сценарийлерге арналған Project Operations_

Нақты мәндер — жоба бойынша аяқталған жұмыс көлемі. Олар уақыт пен шығыс жазбалары, журнал жазбалары мен есеп-шоттар нәтижесінде жасалады.

## <a name="journal-lines-and-time-submission"></a>Журнал жолдары және уақытты ұсыну

Уақыт жазбасы туралы қосымша ақпаратты [Уақыт жазбасын шолу](../time/time-entry-overview.md) бөлімінен қараңыз.

### <a name="time-and-materials"></a>Уақыт және материалдар

Уақыт жазбасы уақыт және материалдар келісім-шарт жолына тағайындалған жобамен байланыстырылған кезде, жүйе екі журнал жолын жасайды, оның біреуі өзіндік құны, екіншісі шот ұсынылмаған сатылым.

### <a name="fixed-price"></a>Бекітілген баға

Бағасы бекітілген келісім-шарт жолына тағайындалған жобаға уақыт жазбасы жіберілген кезде, жүйе журнал жолын тек құн үшін жасайды.

### <a name="default-pricing"></a>Әдепкі баға

Әдеттегі бағаларды жасау логикасы журнал жолында орналасқан. Уақыт жазбасындағы өріс мәндері журнал жолына көшіріледі. Бұл мәндер транзакция күнін, жоба тағайындалған келісім-шарт жолын және тиісті бағатізбедегі валютаны қамтиды.

**Рөл** және **Бөлімше** сияқты әдепкі бағаларды көрсететін өрістер тиісті бағаның журнал жолын анықтау үшін пайдаланылады. Уақыт жазбасына реттелетін өрісті қосуға болады. Өріс мәнін нақты мәндерге тарату қажет болса, нақты мәндер нысанында өрісті жасап, өрісті уақыт жазбасынан нақты мәндер өрісіне көшіру үшін өріс салыстыруларын пайдаланыңыз.

## <a name="journal-lines-and-basic-expense-submission"></a>Журнал жолдары және негізгі шығысты жіберу

Шығыс жазбасы туралы қосымша ақпаратты [Шығыстарды шолу](../expense/expense-overview.md) бөлімінен қараңыз.

### <a name="time-and-materials"></a>Уақыт және материалдар

Негізгі шығыс жазбасы уақыт және материалдар келісім-шарт жолына тағайындалған жобамен байланыстырылған кезде, жүйе екі журнал жолын жасайды, оның біреуі өзіндік құны, екіншісі шот ұсынылмаған сатылым.

### <a name="fixed-price"></a>Бекітілген баға

Бағасы бекітілген келісім-шарт жолына тағайындалған жобаға негізгі шығыс жазбасы жіберілген кезде, жүйе журнал жолын тек құн үшін жасайды.

### <a name="default-pricing"></a>Әдепкі баға

Шығыстар бойынша әдепкі бағаларды енгізу логикасы шығыстар санатына негізделген. Транзакция жасалған күн, жоба тағайындалған келісім-шарт жолы және валюта тиісті бағатізбені анықтау үшін пайдаланылады. Дегенмен, әдепкі бойынша, бағаның өзі үшін пайдаланушының енгізген сомасы қатысты шығыстың құн және сатылым журнал жолдарына тікелей орнатылады.

Шығыс жазбаларында бірліктің санат негізіндегі жазбасының әдепкі бағасы қолжетімді емес.

## <a name="use-entry-journals-to-record-costs"></a>Шығыстарды жазу үшін жазба журналдарын пайдалану

Құнды немесе материалдағы, алымдағы, уақыттағы, шығыстағы табысты немесе салық транзакцияларының кластарын жазу үшін жазба журналдарын пайдалануға болады. Журналдарды келесі мақсаттарда пайдалануға болады:

- Жоба бойынша материалдардың нақты құны мен сатылымдарын жазыңыз.
- Транзакцияның нақты көрсеткіштерін басқа жүйеден Microsoft Dynamics 365 Project Operations жүйесіне ауыстырыңыз.
- Басқа жүйеде орын алған шығындарды жазыңыз. Бұл құндар сатып алу немесе қосымша келісім-шарт құнын қамтуы мүмкін.

> [!IMPORTANT]
> Бағдарлама журнал жолының түрін немесе журнал жолына енгізілген тиісті бағаны тексермейді. Сондықтан, нақты көрсеткіштердің жобаға тигізетін бухгалтерлік есебінің әсерін толық білетін пайдаланушы ғана нақты көрсеткіштер жасау үшін жазба журналдарын пайдалануы керек. Осы журнал түрінің әсеріне байланысты, жазба журналдарын жасауға кімнің рұқсаты бар екенін мұқият таңдау керек.

Төмендегі кестелерде жобаның уақыт және материалдық екеніне, бағасының бекітілгеніне, алдын ала сату кезеңінде екеніне немесе ішкі жоба екеніне байланысты жасалған әртүрлі нақты мәндер көрсетілген.
