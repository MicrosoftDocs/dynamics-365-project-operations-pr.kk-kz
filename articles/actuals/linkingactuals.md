# <a name="transaction-origins---link-actuals-to-their-source"></a>Транзакцияның бастаулары - нақты деректерді олардың көзіне байланыстырыңыз

_**Келесіге қолданылады:** ресурс/биржадан тыс негіздегі сценарийлерге арналған Project Operations, жеңілдетілген орналастыру - проформа-шотын ұсыну мәмілесі_

Транзакцияның шығу жазбалары нақты деректерді олардың көзіне, мысалы, уақыт жазбалары, шығыс жазбалары, материалдарды пайдалану журналдары және жоба шот-фактураларымен байланыстыру үшін жасалады.

Төмендегі мысалда Project Operations жобасының жарамдылық кезеңіндегі уақыт жазбаларының әдеттегі өңделуі көрсетілген.

> ![Жоба операцияларында бүкіл уақытты өңдеу.](media/basic-guide-17.png)
 
1. Уақыт жазбасын жіберу екі журнал жолының жасалуын тудырады: біреуі құн үшін және екіншісі шотсыз сатылымдар үшін.
2. Уақыт жазбасын түпкілікті мақұлдау екі нақтының жасалуын тудырады: біреуі құн үшін және екіншісі шотсыз сатылымдар үшін.
3. Пайдаланушы жоба есеп-шотын жасаған кезде, шот ұсынылмаған сатылымның нақты мәнінен алынған деректерді пайдалану арқылы есеп-шот жолының транзакциясы жасалады.
4. Есеп-шот расталған кезде екі жаңа нақты мән жасалады: шот ұсынылмаған сатылымды кері бағыттау және шот ұсынылған сатылым нақты мәні.

Осы өңдеу жұмыс үрдісіндегі әрбір оқиға уақыт жазбасы, журнал жолы, нақты және шот-фактура жолының мәліметтері бойынша жасалған осы жазбалар арасындағы қарым-қатынастардың ізін құруға көмектесу үшін Транзакцияның бастапқы нысанында жазбаларды жасауды іске қосады.
