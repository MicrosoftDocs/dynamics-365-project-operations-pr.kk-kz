# <a name="transaction-origins---link-actuals-to-their-source"></a>Транзакция көздері - нақты мәндерді олардың дереккөзімен байланыстыру

_**Келесіге қолданылады:** ресурс/биржадан тыс негіздегі сценарийлерге арналған Project Operations, жеңілдетілген орналастыру - проформа-шотын ұсыну мәмілесі_

Транзакция көзінің жазбалары нақты мәндерді олардың дереккөзімен (мысалы, уақыт жазбалары, шығыс жазбалары, материалдарды пайдалану журналдары және жоба есеп-шоттары) байланыстыру үшін жасалады.

Төмендегі мысалда Project Operations жобасының жарамдылық кезеңіндегі уақыт жазбаларының әдеттегі өңделуі көрсетілген.

> ![Project Operations бағдарламасындағы уақыт жазбаларын өңдеу.](media/basic-guide-17.png)
 
1. Уақыт жазбасын жіберу екі журнал жолының жасалуына себеп болады: бір жол құнға арналған, ал екінші жол шот ұсынылмаған сатылымға арналған.
2. Уақыт жазбасын соңғы бекіту екі нақты мәннің жасалуына себеп болады: бір жол құнға арналған, ал екінші жол шот ұсынылмаған сатылымға арналған.
3. Пайдаланушы жоба есеп-шотын жасаған кезде, шот ұсынылмаған сатылымның нақты мәнінен алынған деректерді пайдалану арқылы есеп-шот жолының транзакциясы жасалады.
4. Есеп-шот расталған кезде екі жаңа нақты мән жасалады: шот ұсынылмаған сатылымды кері бағыттау және шот ұсынылған сатылым нақты мәні.

Уақыт жазбасында, журнал жолында, нақты мәнде және есеп-шот жолы мәліметтерінде жасалған осы жазбалардың арасында қарым-қатынас жолын құруға көмектесу үшін, осы өңдеу жұмыс ағынындағы әрбір оқиға транзакция көзінің нысанында жазбалар жасауды іске қосады.
